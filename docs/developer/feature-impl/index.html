機能の実装

このドキュメントでは、WebLogic Deploy Toolingの特定の機能がソース・コードでどのように実装されるかについて説明します。

CreatorクラスとDeployerクラスの階層

ドメイン・モデルのtopologyセクション内での個々のフォルダおよび属性の作成は、モジュールwlsdeploy.tool.create.creator.pyのJythonクラスCreatorのサブクラスを使用して行います。 Creatorクラスには、ドメイン・モデル内のネストされたフォルダを再帰処理し、それらのフォルダを作成または更新し、その属性を設定または更新するためのベース・メソッドが用意されています。 各サブクラスは、これらのメソッドをオーバーライドして、様々な機能領域の動作のバリエーションを考慮できます。

たとえば、SecurityProviderCreatorサブクラスは、メソッド_create_named_subtype_mbeansを特別な処理でオーバーライドして、既存のセキュリティ・プロバイダをすべて削除し、モデル内のデータから再作成します。

ドメイン・モデルのresourcesセクションのフォルダおよび属性の更新も同様のパターンに従いますが、これらのモジュールのベース・クラスはモジュールwlsdeploy.tool.deploy.deployer.pyのDeployerです。

クラスTopologyUpdaterは、Deployerの特別なサブクラスであり、最初の作成後にtopologyセクションの要素を更新するために使用されます。
