WebLogic Deploy Toolのビルド

前提条件

ローカル・ビルド環境には、次のソフトウェアをインストールする必要があります:

  • Oracle WebLogic Serverインストール・バージョン12.2.1以上
  • JDKバージョン8
  • Maven 3以降

WLSTのロケーションの指定

ユニット・テストはWLST内で実行する必要があるため、テストの実行にはWebLogic Serverのインストールが必要です。

WLSTディレクトリは、次のいずれかの方法で指定できます:

  • mvnコマンドラインで-Dunit-test-wlst-dir=<wlst-directory>を指定します。

  • -Dunit-test-wlst-dir=<wlst-directory>値を含む単一行を含むファイル.mvn/maven.configファイルをプロジェクト・ディレクトリに作成します。 .mvnディレクトリには、コピーして開始点として使用できるmaven.config-templateファイルが含まれています。

このような場合、<wlst-directory>はWLSTスクリプト(wlst.shまたはwlst.cmd)への完全修飾パスを参照します。

開発およびビルドにIDEを使用している場合、maven-configファイルを作成すると、IDE内でいくつかのMavenタスクを実行できます。

ビルド・コマンド

プロジェクトを変更する場合は、次のコマンドラインを使用してプロジェクトをビルドできます:

$ mvn -Dunit-test-wlst-dir=<wlst-directory> clean install

これにより、プロジェクト全体がビルドされ、ユニット・テストが実行されます。 前述のようにmaven.configファイルを作成した場合は、-Dunit-test-wlst-dir=引数を省略します。

WLSTディレクトリを指定する別のオプションは、環境変数WLST_DIRを設定することです。 両方を使用する必要はなく、 -D設定が優先されます。

変更せず、最新バージョンのビルドのみに関心がある場合は、次のコマンドラインを使用してユニット・テストをスキップできます:

$ mvn -DskipTests clean install

作成されたインストーラZIPファイルは、WLSDEPLOY_HOME/installer/targetディレクトリの下にあります。
