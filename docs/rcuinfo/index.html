モデルでのRCU接続情報の指定

ドメインの作成または更新時に、コマンド行引数-rcu_dbおよび-rcu_prefixを使用せずにRCUデータベース接続情報を指定するための新しいセクションRCUDbInfoがモデルのdomainInfoセクションにあります。 これを使用して、接続文字列がより複雑で追加のオプションを必要とするデータベースをサポートします。

たとえば、JRFドメインにOracle Autonomous Transaction Processing Cloud Databaseを使用するには、モデルに次の情報を指定します:

domainInfo:
    RCUDbInfo:
        useATP : true
        rcu_prefix : DEV
        rcu_admin_password: <database admin password is required only when you specify -run_rcu flag>
        rcu_schema_password : <RCU schema password>
        atp.admin.user : admin
        tns.alias : dbatp_tp
        javax.net.ssl.keyStorePassword : <atp wallet password>
        javax.net.ssl.trustStorePassword : <atp wallet password>
        oracle.net.tns_admin: <optional: absolute path of the unzipped wallet root directory (outside of the archive), if the wallet.zip is not included in the archive>

データベース・ウォレットは、atpwallet zipentry構造のアーカイブ・ファイルに含めることができます

atpwallet/Walletxyz.zip

または、oracle.net.tns_adminでATPウォレットzipファイルの解凍されたルート・ディレクトリを指定します。

-run_rcuフラグを指定してドメインの作成ツールを使用すると、Oracle Autonomous Transaction Processing Cloudデータベースに対してRCUスキーマが作成され、JRFドメインのデータソースがデータベースを使用するように構成されます。 次に例を示します。 

weblogic-deploy/bin/createDomain.sh -oracle_home /u01/wls12213 -domain_type JRF -domain_home /u01/data/domains/demodomain -archive_file DemoDomain.zip -run_rcu

ATP以外のデータベースの場合は、次の例を使用します:

domainInfo:
    RCUDbInfo:
        rcu_prefix : DEV
        # Optional rcu_db_user for creating RCU schema if -run_rcu flag is specified. Default user is SYS if not specified.
        # This user must have SYSDBA privilege and this is the equivalent of -dbUser in the RCU utility.
        rcu_db_user: superuser
        rcu_schema_password : <rcu schema password, will be prompted if not specified>
        rcu_admin_password : <database admin password is required only when you specify -run_rcu flag, will be prompted
         if not specified>
        rcu_db_conn_string : 'dbhost:1521/pdborcl'

リポジトリ作成ユーティリティのRCU -variablesオプションを、キーrcu_variablesとともにRCUDbInfoセクションに含めることができるようになりました:

domainInfo:
    RCUDbInfo:
        rcu_variables : 'xxxx'

Note: 0.23より前のリリースでは、useATPフラグは0、1、'true'または'false'の値のみを受け入れます。

WDTおよび -run_rcuオプションを使用してドメインを作成する場合、RCUDbInfoセクションで拡張XMLファイルを指定できます。

これは、createRepositoryとdropRepositoryのコマンドライン引数RCU -compInfoXMLLocation <file path> -storageXMLLocation <file path>に関連付けられます

wlsdeploy/rcu/configのロケーションを使用して、XMLファイルをアーカイブ・ファイルに含めます。 次に、この相対的なロケーションをモデルのRCUDbInfoセクションに含めます。 

domainInfo:
    RCUDbInfo:
        compInfoXMLLocation: 'wlsdeploy/rcu/config/MyComponentInfo.xml'
        storageXMLLocation: 'wlsdeploy/rcu/config/MyStorage.xml'
