ドメインの作成

  1. 次のステップでの使用のために:

    • パスワードの作成に必要なルールに従って、ユーザー名とパスワードを選択します(少なくとも1つの数字または特殊文字を含む8文字の英数字)。
    • 出力を書き込むことができるディレクトリを選択または作成します。

  2. create-weblogic-credentialsスクリプトを使用して、ドメインのusernameおよびpasswordを含むWebLogicドメイン管理者資格証明のKubernetesシークレットを作成します:

    $ kubernetes/samples/scripts/create-weblogic-domain-credentials/create-weblogic-credentials.sh \
  -u <username> -p <password> -n sample-domain1-ns -d sample-domain1

    このサンプルでは、domainUIDが-dフラグで指定された値に置き換えられるdomainUID-weblogic-credentialsという名前のシークレットが作成されます。 たとえば、前述のコマンドを実行すると、sample-domain1-weblogic-credentialsという名前のシークレットが作成されます。

  3. create-domainスクリプトを実行して、ドメイン・ホームを含む新しいイメージを作成します。 最初に、サンプルcreate-domain-inputs.yamlファイルをコピーし、コピーを次のように更新します:

    • domainUID: sample-domain1
    • image: スクリプトによって作成される新しいイメージに別の名前を付ける必要がないかぎり、空のままにします。 たとえば、コンテナ・レジストリからイメージをプルする必要があるリモート・クラスタを使用している場合、この値を完全修飾イメージ名に設定する必要があります。 イメージを手動でプッシュする必要があることに注意してください。
    • weblogicCredentialsSecretName: sample-domain1-weblogic-credentials
    • namespace: sample-domain1-ns
    • domainHomeImageBase: container-registry.oracle.com/middleware/weblogic:12.2.1.4

    たとえば、コピーにmy-inputs.yamlという名前を付けたとします:

    $ cd kubernetes/samples/scripts/create-weblogic-domain/domain-home-in-image
$ ./create-domain.sh -i my-inputs.yaml -o /<your output directory> -u <username> -p <password> -e

    -uおよび-pオプションで、ステップ2でKubernetesシークレットを作成したときに指定したものと同じWebLogicドメイン管理者のユーザー名とパスワードをそれぞれ指定する必要があります。

  4. オペレータがドメインのサーバーを起動したことを確認します:

    a. kubectlを使用して、ドメインが作成されたことを示します: 

    $ kubectl describe domain sample-domain1 -n sample-domain1-ns

    b. しばらくすると、管理サーバーと管理対象サーバーが実行されていることがわかります。 

    $ kubectl get pods -n sample-domain1-ns

    c ドメインのすべてのKubernetesサービスも表示されます。 

    $ kubectl get services -n sample-domain1-ns

  5. 「サンプル」 Helmチャートを使用して、ドメイン・ネームスペースにドメインのイングレスを作成します:

    $ helm install sample-domain1-ingress kubernetes/samples/charts/ingress-per-domain \
  --namespace sample-domain1-ns \
  --set wlsDomain.domainUID=sample-domain1 \
  --set traefik.hostname=sample-domain1.org

  6. イングレス・コントローラが新しいイングレスを検出し、ドメインのサーバー・ポッドに正常にルーティングされたことを確認するには、次の例に示すように、HTTP 200ステータス・コードを返すWebLogic ReadyAppフレームワークのURLにリクエストを送信します。

    $ curl -v -H 'host: sample-domain1.org' http://localhost:30305/weblogic/ready
  About to connect() to localhost port 30305 (#0)
    Trying 10.196.1.64...
    Connected to localhost (10.196.1.64) port 30305 (#0)
> GET /weblogic/ HTTP/1.1
> User-Agent: curl/7.29.0
> Accept: */*
> host: domain1.org
>
< HTTP/1.1 200 OK
< Content-Length: 0
< Date: Thu, 20 Dec 2018 14:52:22 GMT
< Vary: Accept-Encoding
<   Connection #0 to host localhost left intact

    Kubernetesクラスタが実行されている場所によっては、ファイアウォール・ポートを開くか、セキュリティ・リストを更新してこのポートへのイングレスを許可する必要があります。

  7. WebLogic Server管理コンソールにアクセスするには:

    a. my-inputs.yamlファイルを編集して(コピーにmy-inputs.yamlという名前を付けたと仮定)、exposedAdminNodePort: trueを設定します。

    b. ブラウザでhttp://localhost:30701を開きます。

    WebLogic Server管理コンソールを使用してサーバーを起動または停止しないでください。 「サーバーの起動と停止」を参照してください。
