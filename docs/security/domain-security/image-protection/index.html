コンテナ・イメージ保護

Oracleは、WebLogicドメイン・ホームを含むコンテナ・イメージをプライベート・レジストリにプライベートとして格納することを強くお薦めします。 すべてのローカル・レジストリに加えて、パブリック・コンテナ・レジストリには「GitHubコンテナ・レジストリ」および「Oracle Cloud Infrastructureレジストリ」 (OCIR)が含まれます。

Domain in Imageイメージの一部であるWebLogicドメイン・ホームには、外部リソース(データ・ソース・パスワードなど)へのアクセスに使用されるキーおよび資格証明を含む、ドメインに関する機密情報が含まれます。 また、イメージを使用して、Kubernetesクラスタの外部にWebLogicドメインを公開する実行中のサーバーを作成するかもしれません。

ノート: Domain in Image ドメイン・ホーム・ソース・タイプは、WebLogic Kubernetes Operatorバージョン4.0では非推奨です。 Oracleでは、必要に応じてDomain on PVまたはModel in Imageを選択することをお薦めします。

プライベート・レジストリにアクセスするKubernetesの設定の詳細は、「ドメイン・イメージにアクセスするためのKubernetesの設定」を参照してください。
