    ネームスペース管理

    オペレータ・ネームスペースの管理に関する考慮事項。

    Cannot pull image

    ドメインが起動せず、`ImagePullBackoff`や`Cannot pull image`などのエラーが表示されます。

    Boot identity not valid

    ドメイン内の1つ以上のWebLogic Serverインスタンスが起動せず、サーバー・ログに次のようなエラーが表示されます: 'ブート・アイデンティティが無効です。

    Domain secret mismatch

    ドメイン内の1つ以上のWebLogic Serverインスタンスが起動せず、ドメイン・リソースのステータスまたはポッド・ログで次のようなエラーが報告されます: Domain secret mismatch.

    ノード・ヒーティングの問題

    オペレータは、起動されるWebLogic Serverインスタンスごとにポッドを作成します。 次に、Kubernetesスケジューラはポッドごとにノードを選択します。 デフォルトのスケジューリング・アルゴリズムでは、必要なコンテナ・イメージがすでにプルされているノードを選択する場合に大幅に重み付けされるため、多くの場合、Kubernetesは同じノード上でWebLogic Serverインスタンスのポッドの多くを実行しますが、他のノードはあまり利用されません。 これは通常、ノード・ヒーティングの問題と呼ばれます。

    高速アプリケーション通知の無効化

    高速アプリケーション通知(FAN)をサポートするには、OracleデータベースでGRID (Oracle Grid Infrastructure)を構成します。 GRIDは通常、Oracle RACデータベースに関連付けられていますが、他の構成でも使用できます。 Oracle Autonomous Database-Serverless (ATP-S)はGRIDを提供しません。

    永続ボリュームに対するOCIファイル・ストレージ(FSS)の使用

    Oracle Container Engine for Kubernetes (OKE)でKubernetesクラスタを実行しており、永続ボリューム用のOCIファイル・ストレージ(FSS)を使用してWebLogicドメイン・ホームを格納する場合、オペレータの永続ボリューム・サンプルに示されているように、ドメインの初回作成時に永続ボリュームのファイル所有権を適切に初期化するには、ファイル・システムの処理を更新する必要があります。

    OCIロード・バランサの使用

    Oracle Container Engine for Kubernetes (OKE)でKubernetesクラスタを実行している場合、Traefikなどのイングレス・コントローラをインストールする(またはそれに加えて)タイプ`LoadBalancer`の`Service`を作成することで、OCIでロード・バランサを自動的にプロビジョニングできます。

    PersistentVolumeClaimへのアクセスの提供

    PersistentVolumeClaimへのアクセス権をインスタンスに付与する必要があります。

    ConfigMapへのアクセスの提供

    ConfigMapへのアクセス権をインスタンスに付与する必要があります。

    外部WebLogicクライアント

    このFAQでは、アプリケーションまたはそのリソースがKubernetesにある場合に、WebLogicアプリケーションがWebLogic JMSリソースまたはEJBリソースにアクセスできるようにする方法について説明しています。 これには、同じKubernetesクラスタ内の異なるKubernetesネームスペース間、異なるKubernetesクラスタ間、およびnon-KubernetesのロケーションとKubernetesの間が含まれます。

    Coherenceの要件

    KubernetesでWebLogicドメイン内またはスタンドアロンのいずれかでCoherenceを実行している場合は、Coherenceがクラスタを形成できるようにするための追加要件がいくつかあります。

    ポッド・メモリーとCPUリソース

    Kubernetesリソースのリクエストおよび制限を構成してコンテナ・メモリーおよびCPU使用率をチューニングし、ドメインYAMLファイルのUSER_MEM_ARGS環境変数を使用してWebLogic JVMヒープ使用率をチューニングします。
