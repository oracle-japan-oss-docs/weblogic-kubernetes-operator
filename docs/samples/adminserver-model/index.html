管理サーバーのカスタマイズ

管理サーバーの構成

ドメインの作成ツールでは、ドメイン・モデルを使用して管理サーバーを構成できます。 これらの例は、いくつかの共通構成をモデルで表す方法を示しています。

デフォルトの管理サーバー構成の使用

ドメインの作成ツールを実行すると、ドメイン・タイプに関連付けられたテンプレートによって、すべての属性のデフォルト値を使用してAdminServerという名前の管理サーバーが自動的に作成されます。 ListenAddressやListenPortなどのこれらの属性、またはSSLやServerStartなどのサブフォルダを変更する必要がない場合は、モデルに何も追加する必要はありません。

管理サーバー構成のカスタマイズ

デフォルトの管理サーバーの構成をカスタマイズするには、AdminServerというデフォルト名のサーバーを追加する必要があります。 管理サーバーの名前は変更しないため、トポロジ・セクションでAdminServerName属性を指定する必要はありません。 この例では、いくつかの属性とサブフォルダを示します: 

topology:
    Server:
        AdminServer:
            ListenPort: 9071
            RestartDelaySeconds: 10
            ListenAddress: 'my-host-1'
            Log:
                FileCount: 9
                LogFileSeverity: Info
                FileMinSize: 5000
            SSL:
                HostnameVerificationIgnored: true
                JSSEEnabled: true
                ListenPort: 9072
                Enabled: true

このタイプの構成で最も一般的な問題は、AdminServerである必要がある場合に、Serverの下のフォルダの名前のスペルを間違えることです。 これにより、デフォルト名で管理サーバーが作成され、スペルが間違った追加の管理対象サーバーが作成されます。

異なる名前での管理サーバーの構成

管理サーバーにデフォルトのAdminServer以外の名前を付ける場合は、AdminServerName属性でその名前を指定し、Serverセクションでその名前を使用する必要があります。 この例では、my-admin-serverという名前を使用します: 

topology:
    AdminServerName: 'my-admin-server'
    Server:
        'my-admin-server':
            ListenPort: 9071
            RestartDelaySeconds: 10
            ListenAddress: 'my-host-1'
            Log:
                FileCount: 9
                LogFileSeverity: Info
                FileMinSize: 5000
            SSL:
                HostnameVerificationIgnored: true
                JSSEEnabled: true
                ListenPort: 9072
                Enabled: true

このタイプの構成で最も一般的な問題は、AdminServerName属性とServerフォルダ内の名前が一致しないことです。 これにより、デフォルトの管理サーバーの名前がAdminServerNameの値に変更され、Serverの下に追加の管理対象サーバーとして作成されるフォルダが変更されます。

ドメインの作成後に管理サーバーの名前を変更することはできないため、AdminServerName属性への変更はドメインの更新ツールによって無視されます。
