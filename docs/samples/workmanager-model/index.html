ワーク・マネージャのモデリング

このWDTドメイン・モデル・サンプル・セクションには、ワーク・マネージャとそれに関連するリクエスト・クラスおよび制約の一般的な構成があります。 これらの要素は、モデルのresourcesセクションのSelfTuningフォルダで構成されます。 

resources:
    SelfTuning:
        Capacity:
            capacity40:
                Target: 'cluster-1'
                Count: 40
        MaxThreadsConstraint:
            threeMax:
                Target: 'cluster-1'
                Count: 3
        MinThreadsConstraint:
            twoMin:
                Target: 'cluster-1'
                Count: 2
        FairShareRequestClass:
            appFairShare:
                Target: 'cluster-1'
                FairShare: 50
            highFairshare:
                Target: 'cluster-1'
                FairShare: 80
            lowFairshare:
                Target: 'cluster-1'
                FairShare: 20
        ResponseTimeRequestClass:
            fiveSecondResponse:
                Target: 'cluster-1'
                GoalMs: 5000
        ContextRequestClass:
            appContextRequest:
                Target: 'cluster-1'
                ContextCase:
                    Case1:
                        GroupName: Administrators
                        RequestClassName: highFairshare
                        Target: 'cluster-1'
                    Case2:
                        UserName: weblogic
                        RequestClassName: lowFairshare
                        Target: 'cluster-1'
        WorkManager:
            myWorkManager:
                Capacity: capacity40
                ContextRequestClass: appContextRequest
                # FairShareRequestClass: appFairShare
                IgnoreStuckThreads: true
                MaxThreadsConstraint: threeMax
                MinThreadsConstraint: twoMin
                # ResponseTimeRequestClass: fiveSecondResponse
                Target: 'cluster-1'

このサンプルでは、FairShareRequestClassおよびResponseTimeRequestClassの割当ては、myWorkManagerにコメントとして含まれています。 ワーク・マネージャで指定できるリクエスト・クラス・タイプは1つのみです。

その他の構成オプションには、追加のサブフォルダおよび属性を使用できます。 これらは、「モデル・ヘルプ・ツール」を使用して決定できます。 たとえば、次のコマンドでは、WorkManagerフォルダの属性とサブフォルダがリストされます: 

$ ${WDT_HOME}/bin/modelHelp.sh -oracle_home /tmp/oracle resources:/WorkManager

このサンプルでは、ターゲット・クラスタcluster-1をこのモデル内の他の場所で定義するか、更新するドメインにすでに存在している必要があります。
