2020年7月17日 v3.0.0 はい。詳細は、「オペレータのアップグレード」を参照してください。 停止時間なしでトポロジおよび構成オーバーライドの変更を適用するためのModel in Image機能およびサポートを追加します。 Helm 2.xのサポートの削除。オペレータのパフォーマンスが向上し、同じKubernetesクラスタ内の多数のドメインを管理できます。
2020年6月22日 v2.6.0 いいえ Kubernetes 1.16、1.17および1.18のサポート。 Kubernetes 1.13以前のサポートの削除。 このリリースは、2.5.0以下のオペレータを使用するか、アップグレード・パスを提供する3.xを使用して、同じクラスタで実行できます。 Kubernetes 1.17.0でのOracle Linux Cloud Native Environment (OLCNE) 1.1の動作保証済サポート。
2020年2月26日 v2.5.0 いいえ Helm 3.xおよびOpenShift 4.3のサポート。 オペレータは、クラスタ・レベルのKubernetes権限を必要としないネームスペース専用モードでインストールできます。 このバージョンはKubernetes 1.16+ではサポートされていません。「前提条件」を確認してください。
2019年11月15日 v2.4.0 いいえ FMWインフラストラクチャ・ドメインおよびポッド変数の置換に関連する様々な問題の修正が含まれます。 オペレータは、WebLogic Deploy Tooling 1.6.0および最新バージョンのKubernetes Java Clientを使用するようになりました。
2019年8月27日 v2.3.0 いいえ Coherenceクラスタのローリング、ポッド・テンプレートおよび追加ポッド・コンテンツのサポート、およびIstioサービス・メッシュでの実行のための試験的なサポートが追加されました。
2019年6月20日 v2.2.1 いいえ オペレータでKubernetes 1.14.0+がサポートされるようになりました。 このリリースは主にバグ修正リリースであり、次の問題を解決します: ドメイン・ホームが永続ボリューム上にあるドメイン内のサーバーは、起動に失敗することがあります。 これらの障害は、完全なドメイン停止後のイントロスペクション・フェーズで発生します。 イントロスペクション・スクリプトでは、関連するエラー条件をより適切に処理できるようになりました。 また、ドメインは、これらのサービスのDNSアドレスが解決可能になるように、まだ実行されていない「Kubernetesサービスの事前作成」 for WebLogic Serversにオプションを提供するようになりました。 これらのサービスは、IPアドレスを持つように非ヘッドレスとして作成されるようになりました。
2019年6月6日 v2.2.0 いいえ FMW Infrastructureドメインのサポートが追加されました。 WebLogic Serverインスタンスはデフォルトで正常に停止され、停止オプションを構成できるようになりました。 オペレータがJDK 11で構築および実行されるようになりました。
2019年4月4日 v2.1 いいえ 顧客は、生成されたポッドにinitおよびsidecarコンテナを追加できます。
2019年3月4日 v2.0.1 いいえ OpenShiftサポートが動作保証されるようになりました。 構成オーバーライド、クラスタ・サービスおよびドメイン・ステータス処理の修正を含む多くのバグ修正。
2019年1月24日 v2.0 はい。1.xリリースとは互換性がありませんが、2.0-rc2とは互換性があります。 最終バージョン番号およびドキュメントの更新。
2019年1月16日 v2.0-rc2 はい スキーマの更新が完了し、様々なバグが修正されました。
2018年12月18日 v2.0-rc1 はい Helmチャートを使用してオペレータがインストールされるようになり、以前のスクリプトが置き換えられました。 オペレータは、永続ボリュームまたはイメージのユースケースで、ドメイン・スキーマの再設計が必要なドメイン・ホームをサポートするようになりました。 ドメイン構成は、構成オーバーライド・テンプレートを使用してオーバーライドできます。 ロード・バランサおよびイングレスを個別に構成できるようになりました。 WebLogicログは、永続ボリュームまたはポッドのログに記録できます。 サーバーのライフ・サイクル・サポートが追加され、生成されたポッドの構成可能性が大幅に向上しました。 最終的なv2.0リリースは、オペレータ・チームが将来のリリースの一部として下位互換性を提供する予定の初期リリースになります。
2018年9月11日 v1.1 いいえ ドキュメントが拡張され、様々なバグが修正されました。
2018年5月7日 v1.0 いいえ 動的クラスタ、Apache HTTP Server、Voyagerイングレス・コントローラ、およびマルチ・ノード環境のNFSストレージにおけるPVのサポートが追加されました。
2018年4月4日 v0.2 はい 多くのKubernetesアーティファクト名およびラベルが変更されました。 また、ドメインのPVおよびPVCを作成するために生成されたYAMLファイルの名前も変更されました。 これらの変更のため、顧客はオペレータとドメインを再作成する必要があります。
2018年3月20日 v0.1 はい いくつかのファイルおよび入力パラメータの名前が変更されました。 これは、オペレータとドメインの作成方法に影響します。 また、生成されたKubernetesアーティファクトも変更されるため、顧客はオペレータおよびドメインを再作成する必要があります。
