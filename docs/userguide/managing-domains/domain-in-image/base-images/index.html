WebLogic Serverイメージの作成または取得

KubernetesでWebLogicドメインを実行するには、コンテナ・イメージが必要です。 次の2つの主なオプションを使用できます:

  • WebLogic Serverバイナリを含むがドメインを含まないイメージを使用
  • WebLogic Serverバイナリとドメイン・ディレクトリの両方を含むイメージを使用します。

最初のオプションを使用する場合は、Oracle Container Registryから標準のWebLogic Serverイメージを取得する必要があります。「Oracle Container Registryからの標準イメージの取得」を参照してください。 このイメージには、「パッチを適用したカスタム・イメージの作成」で説明されているように、適用済の必須パッチがすでに含まれています。 追加のパッチを使用する場合、そのプロセスをカスタマイズして追加のパッチを含めることができます。

イメージ内にドメイン・ディレクトリを含む2番目のオプションを使用する場合は、「イメージ内のドメインを使用したカスタム・イメージの作成」の説明に従って独自のイメージを構築する必要があります。

Oracle Container Registryにアクセスするためのシークレットの設定

このバージョンのオペレータには、WebLogic Server 12.2.1.3.0とパッチ29135930が必要です。標準イメージのcontainer-registry.oracle.com/middleware/weblogic:12.2.1.3には、このパッチが事前に適用されています。 WebLogic Server 12.2.1.4.0のイメージにはパッチは必要ありません。

Kubernetesで、認証を必要とするOracle Container Registry (OCR)からWebLogic Serverイメージを取得するには、レジストリ資格証明を含むKubernetesのシークレットを作成する必要があります。 OCR資格証明でシークレットを作成するには、次のコマンドを発行します: 

$ kubectl create secret docker-registry SECRET_NAME \
  -n NAMESPACE \
  --docker-server=container-registry.oracle.com \
  --docker-username=YOUR_USERNAME \
  --docker-password=YOUR_PASSWORD \
  --docker-email=YOUR_EMAIL

このコマンドでは、大文字のアイテムを適切な値に置き換えます。 SECRET_NAMEは、後のパラメータ・ファイルで必要になります。 NAMESPACEは、最初のドメインがデプロイされるネームスペースと一致する必要があります。一致しない場合、Kubernetesはそれを検出できません。

次の項で説明するように、各Kubernetesワーカー・ノードでイメージを事前に手動でプルすることをお薦めします。 このアプローチを選択した場合、Kubernetesシークレットは必要ありません。

Oracle Container Registryからの標準イメージの取得

Oracle Container Registryには、企業で使用できるライセンスされた商用Oracleソフトウェア製品のイメージが含まれています。 Oracle Registry Serverにアクセスするには、Oracle Single Sign-On (SSO)アカウントが必要です。 Oracle Container Registryには、管理者が認証し、組織で使用するソフトウェアのイメージを選択できるwebインタフェースが用意されています。 webインタフェースを使用するには、Oracleの標準条件および制限事項に同意する必要があります。 Oracleの標準条件および制限を受け入れた後、標準のdocker pullコマンドを使用して、Oracle Container Registryからソフトウェアのイメージをプルできます。

Oracle Container Registryからイメージをプルするには、webブラウザでhttps://container-registry.oracle.comに移動し、Oracle Single Sign-On認証サービスを使用してログインします。 SSO資格証明がまだない場合は、ページ上部にあるSign Inリンクをクリックして作成します。

webインタフェースを使用して、デプロイするOracleソフトウェア・イメージのOracle標準条件および制限を受け入れます。 これらの用語の受入れは、ソフトウェア・イメージをOracle Single Sign-Onログイン資格証明にリンクするデータベースに格納されます。

Oracle Container Registryには、必要なパッチがすでに適用されたWebLogic Server 12.2.1.3.0イメージと、パッチを必要としないOracle WebLogic Server 12.2.1.4.0および14.1.1.0.0イメージが用意されています。

まず、Oracle Container Registryにログインする必要があります:

$ docker login container-registry.oracle.com

その後、次のコマンドを使用してイメージをプルできます:

$ docker pull container-registry.oracle.com/middleware/weblogic:12.2.1.4

必要に応じて、次のことができます:

  • docker run container-registry.oracle.com/middleware/weblogic:12.2.1.4 sh -c 'source $ORACLE_HOME/wlserver/server/bin/setWLSEnv.sh > /dev/null 2>&1 && java weblogic.version'を使用したWLSバージョンの確認

  • docker run container-registry.oracle.com/middleware/weblogic:12.2.1.4 sh -c '$ORACLE_HOME/OPatch/opatch lspatches'によるWLSパッチの確認

このイメージの使用方法の詳細は、Oracle Container Registryを参照してください。

パッチを適用したカスタム・イメージの作成

Oracle WebLogic Server Kubernetes OperatorおよびWebLogic Server 12.2.1.3.0イメージにはパッチ29135930が必要です。 このパッチには、適用する必要がある前提条件パッチがいくつかあります。 WebLogic Server 12.2.1.4.0イメージにパッチは必要ありません。 WebLogic Serverイメージを作成およびカスタマイズし、必要なパッチを適用するには、WebLogic Image Toolを使用します。

イメージ・ツールを使用するには、Setupの手順および「クイック・スタート」ガイドに従ってください。

WebLogic Serverイメージを構築してパッチを適用するには:

  1. Server JREおよびWebLogic Serverインストーラを「cacheコマンド」に追加します。

    $ imagetool cache addInstaller \
--type=jdk \
--version=8u241 \
--path=/home/acmeuser/wls-installers/jre-8u241-linux-x64.tar.gz

$ imagetool cache addInstaller \
--type=wls \
--version=12.2.1.4.0 \
--path=/home/acmeuser/wls-installers/fmw_12.2.1.4.0_wls_Disk1_1of1.zip

  2. 「ツールの作成」を使用してイメージを構築し、パッチを適用します。

    作成ツールでパッチをダウンロードするには、My Oracle Support資格証明を指定する必要があります。

    
$ imagetool create \
--tag=weblogic:12.2.1.3 \
--type=wls \
--version=12.2.1.3.0 \
--jdkVersion=8u241 \
-–patches=29135930_12.2.1.3.0,27117282_12.2.1.3.0 \
--user=username.mycompany.com \
--passwordEnv=MYPWD

  3. ツールによってイメージが作成された後、イメージがローカル・リポジトリにあることを確認します:

    
$ docker images

イメージ内のドメインを使用したカスタム・イメージの作成

イメージ内のWebLogicドメインを使用してイメージを作成することもできます。 次のいずれかを使用してイメージを作成する方法を示す「サンプル」が提供されています:

  • ドメインを定義するためのWLST
  • 「WebLogicデプロイ・ツール」を使用してドメインを定義します。

これらのサンプルでは、ベース・イメージまたはFROMイメージへの参照が示されています。 必須パッチがインストールされたイメージをこのベース・イメージとして使用する必要があります。 このイメージは、標準のcontainer-registry.oracle.com/middleware/weblogic:12.2.1.3事前パッチ・イメージ、または前述の手順を使用して作成したイメージのいずれかです。 WebLogic Server 12.2.1.4.0イメージにはパッチは必要ありません。

Oracleでは、WebLogicドメインを含むイメージをプライベート・リポジトリに保持することをお薦めします。
