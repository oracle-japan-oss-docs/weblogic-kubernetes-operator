WLSTの使用

WebLogic Scripting Tool (WLST)を使用して、Kubernetesで実行中のドメインを管理できます。 ドメインの作成時にexposeAdminT3Channel設定を使用してT3チャネルを公開するように管理サーバーが構成されている場合、一致するT3サービスを使用して接続できます。 たとえば、domainUIDがdomain1で、管理サーバー名がadmin-serverの場合、サービスは次のように呼び出されます: 

domain1-admin-server-ext

このサービスは、ドメインと同じネームスペースにあります。 外部ポート番号は、このサービスのnodePortを確認することで取得できます: 

$ kubectl get service domain1-admin-server-ext -n domain1 -o jsonpath='{.spec.ports[0].nodePort}'
30012

この例では、nodePortは30012です。 Kubernetesサーバーのアドレスがkubernetes001の場合、WLSTは次に示すようにt3://kubernetes001:30012に接続できます: 

$ ~/wls/oracle_common/common/bin/wlst.sh

Initializing WebLogic Scripting Tool (WLST) ...

Welcome to WebLogic Server Administration Scripting Shell

Type help() for help on available commands

wls:/offline> connect('weblogic','*password*','t3:// kubernetes001:30012')
Connecting to t3:// kubernetes001:30012 with userid weblogic ...
Successfully connected to Admin Server "admin-server" that belongs to domain "base_domain".

Warning: An insecure protocol was used to connect to the server.
To ensure on-the-wire security, the SSL port or Admin port should be used instead.

wls:/base_domain/serverConfig/> exit()


Exiting WebLogic Scripting Tool.
