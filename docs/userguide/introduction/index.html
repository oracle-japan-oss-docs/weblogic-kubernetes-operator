    スタート・ガイド

    オペレータの前提条件およびサポートされている環境を確認します。

    デモ

    WebLogic Server Kubernetes Operatorのビデオ・デモをご覧ください。

    アーキテクチャ

    オペレータは複数の部分で構成されています: オペレータ・ランタイム、Kubernetesカスタム・リソース定義(CRD)のモデル、オペレータをインストールするためのHelmチャート、Kubernetesで実行するためのWebLogicドメインを準備またはパッケージ化するための様々なサンプル・シェル・スクリプト、およびKubernetesクラスタの外部にWebLogicエンドポイントを条件付きで公開するためのサンプルHelmチャートまたはシェル・スクリプト。

    設計の方針

    Oracle WebLogic Server Kubernetes Operator (“オペレータ”)は、従来のデータ・センターのデプロイメントで人間のオペレータが果たすロールと同様のロールを果たすように設計されています。 これには、ドメインで様々なライフ・サイクル操作を正しく実行する方法に関する有用な組込みナレッジのセットが含まれています。

このガイドでは、Oracle WebLogic Server Kubernetes Operatorの詳細なユーザー情報について説明します。 Kubernetesクラスタにオペレータをインストールする方法、およびオペレータを使用してWebLogicドメインを管理する方法について説明します。 オペレータの設計、実装、構築などの方法の詳細は、「開発者ガイド」を参照してください。

重要な用語

このドキュメントでは、特定の意味を持つことを目的としたいくつかの重要な用語を使用します。

用語 定義
クラスタ この用語はあいまいであるため、クラスタのタイプを示すプレフィクスが付きます。 WebLogicクラスタは、一部のアプリケーションまたはコンポーネントをまとめてホストし、それらの間で負荷および状態を共有できる管理対象サーバーのグループです。 Kubernetesクラスタは、ポッドやサービスなどのKubernetesリソースをホストするすべてのマシン(ノード)のグループで、外部ユーザーには単一のエンティティとして表示されます。 「クラスタ」という用語にプレフィクスが付いていない場合は、Kubernetesクラスタであると想定する必要があります。
ドメイン WebLogicドメインは、関連するアプリケーションおよびリソースのグループと、それらを実行するために必要な構成情報です。
イングレス Kubernetes Ingressは、Kubernetes環境のアプリケーションおよびサービスへのアクセスを外部クライアントに提供します。 イングレスは、ロード・バランシングなどの追加機能を提供する場合もあります。
ネームスペース Kubernetesネームスペースは、ポッドやサービスなどの関連オブジェクトをグループ化するために使用できる名前付きエンティティです。
オペレータ Kubernetesオペレータは、複雑なアプリケーションの管理を実行するソフトウェアです。
ポッド Kubernetesポッドは、1つ以上のコンテナを含み、webサーバーやデータベースなどのアプリケーション・コンポーネントのインスタンスの実行環境を提供するオブジェクトです。
ジョブ Kubernetesジョブは、特定のタスクを完了するために実行される1つ以上のポッドを作成するコントローラのタイプです。
シークレット Kubernetesのシークレットは、ユーザー名、パスワード、X.509証明書、その他の任意のデータなどのシークレット情報を格納できる名前付きオブジェクトです。
サービス Kubernetesサービスは、ポッド内のアプリケーション・エンドポイントを他のポッドまたはKubernetesクラスタ外に公開します。 サービスでは、ロード・バランシングなどの追加機能も提供されます。

追加情報

オペレータを使用する前に、「設計理念」を読んでオペレータの設計を理解し、「アーキテクチャの概要」を読んで、オペレータを使用してKubernetesにWebLogicドメインをデプロイする方法など、そのアーキテクチャを理解することをお薦めします。 また、オペレータが必要とする「Kubernetes RBAC定義」の詳細を読む価値もあります。
