ドメインの更新ツール

ドメインの更新ツールは、モデル、アーカイブおよびWLSTを使用して、既存のWebLogic Serverドメインの構成を更新し、WLSTオンライン・モードまたはオフライン・モードでアプリケーションおよびリソースをドメインにデプロイします。 更新ツールは、モデルのtopologyセクションから要素を追加または再構成し、Deploy Applications Toolで説明されているように、resourcesおよびappDeploymentsセクションからアプリケーションおよびリソースをデプロイします。

ドメインの更新ツールは、指定されたモデルの要素のみを追加または更新します。 以前のモデルに存在していた欠落している要素は削除されません。

WLSTオフライン・モードでのドメインの更新ツールの実行は、ドメインの作成ツールの実行によく似ています。必要に応じて、ドメインのロケーションとアーカイブ・ファイル、および個別のモデルと変数ファイルを指定します。 次に例を示します。 

$ weblogic-deploy\bin\updateDomain.cmd -oracle_home c:\wls12213 -domain_type WLS -domain_home domains\DemoDomain -archive_file DemoDomain.zip -model_file DemoDomain.yaml -variable_file DemoDomain.properties

WLSTオンライン・モードでは、WebLogic Server管理サーバーへの接続方法に関する情報を追加します。次に例を示します:

$ weblogic-deploy\bin\updateDomain.cmd -oracle_home c:\wls12213 -domain_type WLS -domain_home domains\DemoDomain -archive_file DemoDomain.zip -model_file DemoDomain.yaml -variable_file DemoDomain.properties -admin_url t3://127.0.0.1:7001 -admin_user weblogic

通常どおり、ツールによってパスワードの入力が求められます(ツールの標準入力にパイピングして指定することもできます)。

ドメインの作成ツールとは異なり、ドメインはすでに確立されているため、ドメインの親ディレクトリではなく完全なドメイン・ホーム・ディレクトリが指定されます。

ドメインの更新ツールは、RCUを使用するドメイン・タイプに対して、RCUデータベースのスキーマの再作成または追加を試行しません。

WLSTオンライン・モードでツールを実行する場合、更新操作では、変更を取得するためにサーバーの再起動またはドメインの再起動が必要になることがあります。 更新操作では、ドメインが再起動されるまで操作を完了できない場合もあります。 これらの条件を、ドメインの更新ツールを呼び出している可能性のあるスクリプトに伝達するために、シェル・スクリプトには、これらの状態を伝達するためのゼロ以外の特殊な終了コードが2つあります:

  • 103 - ドメイン全体を再起動する必要があります。
  • 104 - モデルの変更にはドメインの再起動が必要で、 -cancel_changes_if_restart_requiredが指定されているため、ドメインの変更は取り消されました。

暗号化モデルの使用

WDT暗号化ツールで暗号化されたパスワードがモデルまたは変数ファイルに含まれている場合、コマンドラインで-use_encryptionフラグを使用して作成中にパスワードを復号化し、暗号化が使用されていることを更新ドメイン・ツールに伝え、暗号化パスフレーズの入力を求めます。 データベース・パスワードと同様に、ツールは標準入力(stdinなど)からパスフレーズを読み取り、ユーザー入力なしでツールを実行できるようにすることもできます。

複数のモデルの使用

ドメインの更新ツールは、「複数のモデルの使用」で説明されているように、複数のモデルの使用をサポートしています。
