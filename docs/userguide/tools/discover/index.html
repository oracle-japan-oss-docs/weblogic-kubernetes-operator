ドメインの検出ツール

ドメインの検出ツールは、既存のドメインを検査し、そこから構成およびバイナリを収集することで、モデルおよびアーカイブ・ファイルを作成するブートストラップ・メカニズムを提供します。 ツールによって生成されたモデル・ファイルは、ドメインの作成ツールまたはDeploy Applications Toolでは直接使用できません。これは、ドメインの検出ツールでは既存のドメインからパスワードが検出されないためです。 かわりに、検出したパスワードの--FIX ME--プレースホルダーを配置します。 ドメイン・ユーザーも検出できないため、ツールによってdomainInfoセクションのAdminUserNameおよびAdminPasswordフィールドに同じプレースホルダー値が挿入されます。 このツールの考え方は、生成されたモデルおよびアーカイブ・ファイルを、他のツールを実行するためのニーズにあわせて編集できる開始ポイントを提供することです。

ドメインの検出ツールを実行するには、Oracleホームとドメインのロケーションを指定します。 検出されたドメイン情報が書き込まれるモデル・ファイルおよびアーカイブ・ファイルのロケーションとファイル名を指定します。 モデルはYAMLまたはJSON形式で生成できます。 ファイル名に正しいサフィクスを付けてください。 shスクリプトとcmdスクリプトの両方が提供されています。

基本discoverDomainの例:

$ weblogic-deploy\bin\discoverDomain.cmd -oracle_home c:\wls12213 -domain_home domains\DemoDomain -archive_file DiscoveredDemoDomain.zip -model_file DiscoveredDemoDomain.yaml

アーカイブの作成時に、ツールは、次の点に注意して、ドメイン構成によって参照されるすべてのバイナリ、スクリプトおよび必要なディレクトリの収集を試みます。

  1. ORACLE_HOMEから参照されるバイナリは、モデルドリブン操作が適用されるターゲット・ドメインに存在するとみなされるため、収集されません。 これは、モデルをWebLogic Serverバージョンに依存しないようにするためのキーです。
  2. 現在の形式では、ドメインの検出ツールはローカル・マシンからアクセス可能なバイナリおよびスクリプトのみを収集します。 検出できないバイナリまたはスクリプトに対して警告が生成されますが、可能な場合はそれらのバイナリの構成が収集されます。 欠落しているファイルを適切なロケーションにあるアーカイブに追加し、必要に応じて、アーカイブ内の相対パスを使用してそれらのファイルを指すようにモデルを編集するのは、ユーザーの責任です(たとえば、wlsdeploy/applications/myapp.ear)。
  3. モデル・ファイルを検査する場合は、アーカイブ・ファイルを生成せずに検出ツールを実行できます。 ドメインの作成または更新には、ドメインにインストールされるバイナリ、スクリプトまたはディレクトリの有効なアーカイブ・ファイルが必要です。

変数ファイルのロケーションと名前を指定して、モデルで生成されるパスワード属性の内容をカスタマイズできます。 このファイルは、モデルで見つかった各パスワードのkey=valueを含むテキスト・プロパティ・ファイルです。 キーはパスワード属性の一意のトークン名で、値は置換値(この場合は空の文字列)です。 モデルの属性には、トークン名とプロパティ・フィールド表記が挿入されます。 たとえば、@@PROP:AdminUserName@@や@@PROP:JDBCSystemResource.<Name>.JdbcResource.JDBCDriverParams.PasswordEncrypted@@などです。

変数ファイル名を含むコマンドラインの例:

$ weblogic-deploy\bin\discoverDomain.cmd -oracle_home c:\wls12213 -domain_home domains\DemoDomain -archive_file DiscoveredDemoDomain.zip -model_file DiscoveredDemoDomain.json -variable_file DiscoverDemoDomain.properties

オンラインWLSTを使用してドメインを検出するには、管理ユーザー名と管理URLをコマンドラインに含めます。 STDINに入力するパスワードの入力を求めるプロンプトが表示されます。

オンラインWLSTモードでの実行の例:

$ weblogic-deploy/bin/discoverDomain.sh -oracle_home /u01/oracle -domain_home /u01/oracle/domains/DemoDomain -archive_file ./DiscoveredDemoDomain.zip -model_file ./DiscoveredDemoDomain.yaml -admin_user weblogic -admin_url t3://localhost:7001

対応するバイナリをアーカイブ・ファイルに正常に収集するには、ドメイン・バイナリが配置されているシステムと同じシステムでコマンドを実行する必要があります。

カスタム・テンプレートまたは製品テンプレートを使用してドメインを作成すると、テンプレートによって、モデルの検出やアーカイブへの収集を必要としないリソースがドメインにインストールされます。 ドメインtypedefファイルに対応するドメイン・タイプ引数は、これらのリソースおよびファイルを無視するために、ドメインのタイプを記述する必要があります。 デフォルトでは、discoverはドメイン・タイプWLSを使用して実行されます。これは、WebLogic Serverテンプレートのみがドメインに適用されていることを前提としています。 このツールには、他のドメイン・タイプ(RestrictedJRFおよびJRF)のtypedefが事前に用意されています。 これらのドメイン・タイプまたは別のカスタムtypedefを使用できます。 詳細は、「ドメイン・タイプ定義」を参照してください。

ドメイン型引数の使用例:

$ weblogic-deploy\bin\discoverDomain.cmd -oracle_home c:\wls12213 -domain_home domains\DemoDomain -archive_file DiscoveredDemoDomain.zip -model_file DiscoveredDemoDomain.yaml -domain_type RestrictedJRF

モデルがモデル・ファイルに永続化される前に、変数インジェクタまたはモデル・フィルタがこの順序で実行されます。 最後のステップは検証で、モデル、アーカイブおよび変数ファイルの内容を検証します。 検証が成功すると、モデルは永続化されます。 これら3つのトピックの詳細は、次を参照してください:

  • 変数インジェクション
  • モデル・フィルタ
  • モデルの検証ツール

結果のモデルは、Oracle WebLogic Server Kubernetes Operatorなどの特定のターゲット環境との互換性のために変更することもできます。 詳細は、「ターゲット環境」を参照してください。

問題(または成功)は検出ツールのサマリーに一覧表示されます。 サマリーには、ツールとOracleホームのバージョン、およびツールが実行されたWLSTモード(オンラインまたはオフライン)が出力されます。 すべての警告および重大なメッセージの要約が合計とともにリストされます。

WARNINGメッセージが表示されたサマリーの例:

Issue Log for discoverDomain version 1.5.2-SNAPSHOT running WebLogic version 10.3.6.0 offline mode:

WARNING Messages:

        1. WLSDPLY-06200: Unable to get the Security Realm Provider name in version 10.3.6.0 with offline wlst. The SecurityConfiguration will not be added to the model. The work-around is to manually add the Security Configuration to the model or to discover the domain in online mode : Invalid Security Provider name "Provider" found for provider type at location /SecurityConfiguration/Realm/Adjudicator.

Total:       WARNING :     1    SEVERE :     0

環境変数

次の環境変数を設定できます。

  • JAVA_HOME JDKのロケーション。 これは、有効なJava 7以上のJDKである必要があります。
  • WLSDEPLOY_HOME WebLogic Deploy Toolingインストールのロケーション。 デフォルトでは、ロケーションはdiscoverDomainスクリプトのロケーションから計算されます。
  • WLSTに渡されるWLSDEPLOY_PROPERTIESシステム・プロパティ。

ドメインの検出に対する問題のオープン

GitHubの問題にSTDOUTおよびSTDERRのログ・ストリームを提供してください。 サマリーがリストされていない場合(未処理の例外スタック・トレースが発生した場合)は、OracleおよびWDTのインストール・バージョンと、ツールがオンラインまたはオフラインのどちらのWLSTモードで実行されたかを確認してください。 可能な場合は、モデル、変数およびアーカイブ・ファイルと、ログ・ファイルdiscoverDomain.logを<install home>\weblogic-deploy\logのロケーションから指定します。
