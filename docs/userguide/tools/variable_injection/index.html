変数インジェクタ・ツール

変数インジェクタ・ツールは、変数を含むモデルをトークン化するために使用します。 これらの変数の値は、外部プロパティ・ファイルを使用して割り当てられます。 これにより、同じドメイン・モデルを使用して、異なる環境で新しいドメインを簡単に作成できます。 変数インジェクタ・ツールは、ドメインの検出ツールのオプションとして実行することも、スタンドアロンのコマンドライン・インタフェースから実行することもできます。

ドメイン検出ツールの実行中に変数インジェクタ・ツールを有効にするには、事前定義済のキーワードまたはファイルのCUSTOMリスト(あるいはその両方)を使用して、model_variable_injector.jsonという名前のJSONファイルを<WLSDEPLOY>/libディレクトリに配置することにより、変数インジェクタ構成を作成します。

別のオプションは、「カスタム構成」ディレクトリに変数インジェクションを構成することです。 $WDT_CUSTOM_CONFIGディレクトリにmodel_variable_injector.jsonファイルを作成します。

キーワードはインジェクタ・ディレクティブ・ファイルを指します。 このツールは、ディレクティブをモデル内の属性に適用し、ディレクティブが属性と一致する場合、一意の変数名を持つプロパティ・トークンがモデルに挿入され、属性値が置換されます。 変数名およびモデル属性値は、外部変数プロパティ・ファイルに配置されます。

属性の変数インジェクションは一度のみ実行されます。 propertyトークンは、後続の一致で置換されません。

変数インジェクションが有効な場合、ドメインの検出ツールは、モデルが検出された後、すべてのフィルタが実行された後、かつモデル検証の前に変数インジェクタを呼び出します。

サポートされているキーワードは次のとおりです:

  • CREDENTIALS - すべてのMBean資格証明属性値(ユーザーおよびパスワード)に変数が挿入されます。
  • HOST - モデル内のすべてのMBeanホスト属性値に変数が挿入されます。
  • PORT - モデル内のすべてのMBeanポート属性値に変数が挿入されます。
  • TARGET - モデル内のすべてのMBeanターゲット属性値に変数が挿入されます。
  • TOPOLOGY - モデルのトポロジ・セクションにある共通の環境MBean属性には、変数が挿入されます。 これには、サーバー、マシンおよびノード・マネージャのポート、資格証明およびリスニング・アドレス、クラスタ・メッセージング・モード、アドレスおよびポートが含まれます。
  • URL - モデル内のすべてのMBean URL属性値に変数が挿入されます。

各事前定義済キーワードで使用されるディレクティブは、<WLSDEPLOY>/lib/injectorsフォルダにあるインジェクタJSONファイルで定義されます。 これらのファイルは変更しないでくださいが、そのまま使用できます。

PORTキーワードを使用したmodel_variable_injector.jsonファイルの例を次に示します。

{
	"PORT": {}
}

次に、ポート属性に注入された変数を示すモデル・スニペットを示します。


topology:
    Name: soa_domain
    AdminServerName: AdminServer
    Cluster:
        soa_cluster:
        osb_cluster:
    Server:
        AdminServer:
            ListenAddress: myadmin.example.com
            ListenPort: @@PROP:Server.AdminServer.ListenPort@@
            Machine: machine1
            SSL:
                Enabled: true
                ListenPort: @@PROP:Server.SSL.AdminServer.ListenPort@@
        soa_server1:
            ListenAddress: managed1.example.com
            ListenPort: @@PROP:Server.soa_server1.ListenPort@@
            Cluster: soa_cluster
            Machine: machine2
            SSL:
                Enabled: true
                ListenPort: @@PROP:Server.SSL.soa_server1.ListenPort@@
        soa_server2:
            ListenAddress: managed2.example.com
            ListenPort: @@PROP:Server.soa_server2.ListenPort@@
            Cluster: soa_cluster
            Machine: machine3
            SSL:
                Enabled: true
                ListenPort: @@PROP:Server.SSL.soa_server2.ListenPort@@

生成される変数プロパティ・ファイルは次のとおりです:

Server.AdminServer.ListenPort=7001
Server.AdminServer.SSL.ListenPort=7002
Server.soa_server1.ListenPort=8001
Server.soa_server1.SSL.ListenPort=8002
Server.soa_server2.ListenPort=8001
Server.soa_server2.SSL.ListenPort=8002

ドメインの検出ツールの変数プロパティ・ファイルの名前とロケーションを指定するには、コマンドラインで引数-variable_properties_fileを使用します。 <WLSDEPLOY>/libディレクトリにモデル変数インジェクタ・ファイルが存在しない状態でvariable_properties_file引数を使用すると、エラー状態が発生し、ツールが終了します。 モデル変数インジェクタ・ファイルがディレクトリに存在するが、コマンドライン引数が使用されていない場合、変数プロパティ・ファイルは次のデフォルトで作成されます:

  • model_fileコマンドライン引数がドメインの検出ツールの実行で使用される場合、プロパティ・ファイルの名前とロケーションはモデル・ファイルと同じになり、ファイル拡張子は.propertiesになります。
  • アーカイブ・ファイル引数のみが存在する場合は、アーカイブ・ファイル名とロケーションが使用されます。

アーカイブおよびモデル・ファイルと同様に、ドメインの検出ツールを実行するたびに、既存の変数プロパティ・ファイルの内容が現在の実行の値で上書きされます。

カスタム変数インジェクタ

カスタム・インジェクタ・ディレクティブを指定するには、model_variable_injector.jsonファイルでCUSTOMキーワードを使用します。 CUSTOMキーワードには、カスタム・インジェクタ・ディレクティブJSONファイルのリストが必要です。

インジェクタ・ディレクティブには、トークン化する属性を識別するキーと、オプションのディレクティブ・プロパティのセットが含まれます。 キーは、モデルで定義されている期間で区切られたMBean階層および属性名です。 モデル・セクションの名前は、常にインジェクタ・キーから除外してください。

たとえば、サーバーSSLリスニング・ポートのインジェクタ・キーは次のとおりです。 このディレクティブには、追加プロパティは含まれていません。 

{
  "Server.SSL.ListenPort": {}
}

NOTE: ListenPort属性のモデル内のMBeansの階層。 AdminServerのMBean名はディレクティブに含まれないことに注意してください: 

topology:
    Server:
        AdminServer:
            ListenAddress: myadmin.example.com
            ListenPort: 7001
            Machine: machine1
            SSL:
                Enabled: true
                ListenPort: 7002

これらのカスタム・インジェクタJSONファイルは、変数インジェクタ・ツールによってキーワードの前に処理され、各ファイルはリスト順に処理されます。 属性に注入されたプロパティは、後続の一致で置換されません。

カスタム・ディレクティブのプロパティ

次のプロパティを含めて、指定に従ってディレクティブを絞り込みます。

  • force:<attribute> MBean階層がモデルに存在するが、属性が存在しない場合、属性は検出されたモデルに追加され、永続化されます。 モデルに格納される値は、WebLogicのデフォルト値です。

  • variable_value: 変数プロパティで、モデル値を指定された値に置き換えます。 これはforceディレクティブと組み合わせて使用でき、デフォルト値を指定された値に置き換えます。

  • regexp: モデル内の属性の文字列値またはマップ値に適用されるregexpパターンのリスト。 パターンが一致すると、文字列またはディクショナリの一致部分にプロパティ・トークンと一意の変数名が挿入されます。

    • pattern: 属性の文字列値またはマップ値に適用する正規表現パターン。

    • suffix: 一意の変数名を作成するために結果の各変数名に追加するサフィクス名。

regexpリストは、文字列値またはマップのセグメントのみをトークン化する必要がある場合に便利です(変数プロパティ・ファイル内のプロパティ値のクリーンなリストを提供します)。 複数のトークンを文字列に注入したり、複数のパターンを使用してマップすることができます。 ただし、パターンが複数ある場合は、それぞれにサフィクスを指定する必要があります。 これにより、文字列またはマップ内のトークンごとに一意の変数名を生成できます。

次に、regexpディレクティブ・リストを使用して文字列値内のセグメントを効率的に検索する方法の例を示します。 この例では、特別なOracle URL表記法を使用する各Oracle JDBC URLでホストとポートを検索し、変数プロパティ・ファイルにホストとポートのエントリを作成します。

このモデルでは、次のようなURLが見つかることを想定しています:

    JDBCSystemResource:
        Database1:
            JdbcResource:
                JDBCDriverParams:
                    URL: 'jdbc:oracle:thin:@(DESCRIPTION=(ADDRESS_LIST=(ADDRESS=(PROTOCOL=TCP)(HOST=slc05til.us.oracle.com)(PORT=1521)))(CONNECT_DATA=(SERVICE_NAME=orcl.us.oracle.com)))'

カスタム・インジェクタJSONファイルにディレクティブを作成します:

  "JDBCSystemResource.JdbcResource.JDBCDriverParams.URL":
  {
    "regexp": [
      {
        "pattern": "(?<=PORT=)[\\w.-]+(?=\\))",
        "suffix": "Port"
      },
      {
        "pattern": "(?<=HOST=)[\\w.-]+(?=\\))",
        "suffix": "Host"
      }
    ]
  },

ドメインの検出ツールの実行中に、パターンがURL文字列に適用され、文字列にトークンが挿入されます:

URL: 'jdbc:oracle:thin:@(DESCRIPTION=(ADDRESS_LIST=(ADDRESS=(PROTOCOL=TCP)(HOST=@@PROP:JDBCSystemResource.Database1.JdbcResource.JDBCDriverParams.URL--Host@@:)(PORT=@@PROP:JDBCSystemResource.Database1.JdbcResource.JDBCDriverParams.URL--Port@@)))(CONNECT_DATA=(SERVICE_NAME=orcl.us.oracle.com)))'

プロパティ・ファイルに格納されている変数は次のとおりです:

JDBCSystemResource.Database1.JdbcResource.JDBCDriverParams.URL--Host=slc05til.us.oracle.com
JDBCSystemResource.Database1.JdbcResource.JDBCDriverParams.URL--Port=1521

変数インジェクション用の特定のMBean名の選択

この最後のカスタム・ディレクティブを使用すると、プロパティを注入するモデル内のMBeanの名前付きエントリを明示的に定義できます。 たとえば、特定のサーバーのみの属性をトークン化できます。 インジェクタ・ディレクティブ階層内の特定のMBeanに1つ以上の名前のリストを定義するには、次のようにリストを書式設定します: 

MBean[comma separated list of names]

サーバー・ディレクティブにAdminServerという名前の管理サーバーのみを選択するには、Server[AdminServer]の形式を使用します。 サーバーsoa_server1およびsoa_server2を選択するには、キーをServer[soa_server1,soa_server2]としてフォーマットします。

インジェクタ・ツールは、ユーザー・リスト、MANAGED_SERVERS (モデル内のすべての管理対象サーバー)およびADMIN_SERVER (モデル内の管理サーバー)に対して2つのKEYWORDSを認識します。

管理サーバーのSSLリスニング・ポートのカスタム・インジェクタは次のとおりです:

{
  "Server[ADMIN_SERVER].SSL.ListenPort": {},
}
