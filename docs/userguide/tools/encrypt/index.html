モデルの暗号化ツール

Oracleのセキュリティ標準を満たすために、暗号化アルゴリズムを実行するにはJDK 8が必要です。

モデルには、WebLogic Serverドメイン構成が含まれます。 特定のタイプのリソースおよびその他の構成では、パスワードが必要です。たとえば、JDBCデータ・ソースでは、データベース接続を確立するユーザーのパスワードが必要です。 パスワードを必要とするリソースを作成または構成する場合は、そのパスワードをモデルに直接指定するか、変数ファイルに指定する必要があります。 ク・リア・テキスト・パスワードは、構成をソースとして格納することには導電性がないため、モデルの暗号化ツールを使用すると、モデル作成者は、パスフレーズ・ベースの可逆暗号化を使用してモデルおよび変数ファイルのパスワードを暗号化できます。 暗号化されたパスワードを含むモデルでツールを使用する場合、ツールが必要なWebLogic Server構成(ドメイン固有のキーに基づく独自の暗号化メカニズムをサポート)を設定するためにメモリー内のパスワードを復号化できるように、暗号化パスフレーズを指定する必要があります。 WebLogic Deploy Tooling暗号化メカニズムを使用する必要はありませんが、クリア・テキスト・パスワードをディスクに格納することはお薦めしないため、強くお薦めします。

作成、更新およびデプロイ・ツールでは、一連のモデルを使用できます。 暗号化モデルは、モデルのセットを暗号化します。 各モデルは、同じパスフレーズを使用して暗号化され、元のロケーションに書き戻されます。

WebLogic Deploy Toolでは、ドメインで暗号化されたパスワードをモデル内で直接使用することもできます。 モデルの暗号化ツールは、このメソッドと組み合せて使用しないでください。

モデルで暗号化ツールを実行するには、次のコマンドを実行します:

$ weblogic-deploy\bin\encryptModel.cmd -oracle_home c:\wls12213 -model_file UnencryptedDemoDomain.yaml
