モデル・ヘルプ・ツール

モデル・ヘルプ・ツールは、ドメイン・モデルのセクションおよびフォルダに有効なフォルダおよび属性に関する情報を提供します。 これは、新しいドメイン・モデルを作成する場合や、検出されたモデルを含む既存のモデルを拡張する場合に便利です。

モデル・ヘルプ・ツールは、WebLogic Deploy Tooling 1.8の新機能です。 -model_sample引数は、モデル・サンプルがデフォルトの出力形式になったときに、WebLogic Deploy Tooling 1.9.2から非推奨になりました。

モデル・ヘルプ・ツールを使用した簡単な例を次に示します:

$ <wls-deploy-home>/bin/modelHelp.sh -oracle_home /tmp/oracle resources:/JDBCSystemResource

この結果、次のように出力されます。

Attributes and sub-folders for resources:/JDBCSystemResource

resources:
    JDBCSystemResource:
        'JDBC-1':
            CompatibilityName:       # string
            DeploymentOrder:         # integer
            DeploymentPrincipalName: # string
            DescriptorFileName:      # string
            ModuleType:              # string
            Notes:                   # string
            SourcePath:              # string
            Target:                  # delimited_string

            JdbcResource:
                # see /JDBCSystemResource/JdbcResource

            SubDeployment:
                'SubDeployment-1':
                    # see /JDBCSystemResource/SubDeployment

この出力は、モデルのresourcesセクションのJDBCSystemResourceフォルダで使用可能な8つの属性と2つのサブフォルダを示しています。 各属性には、追加する値のタイプを説明するコメントが含まれます。

複数のインスタンスをサポートするフォルダ(この例ではJDBCSystemResourceなど)は、'JDBC-1'などの導出名で表示されます。

各サブフォルダには、そのサブフォルダに関する追加情報の表示に使用できるモデル・パスを含むコメントが含まれています。 たとえば、'SubDeployment-1'の属性およびサブフォルダを決定するには、モデル・ヘルプ・ツールをコメントからのモデル・パスを使用して再起動できます: 

$ <wls-deploy-home>/bin/modelHelp.sh -oracle_home /tmp/oracle -model_sample /JDBCSystemResource/SubDeployment

前述の情報を使用して、このモデル・セクションを構築できます:

resources:
    JDBCSystemResource:
        CompatibilityName: 'myName'
        DeploymentOrder: 5
        Target: 'ms1,ms2'
        'JDBC-1':
            # JdbcSystemResource attributes and folders
        SubDeployment:
            deployment1:
                # SubDeployment attributes and folders
            deployment2:
                # SubDeployment attributes and folders

サンプル・モデルから要素をコピーして完全なドメイン・モデルを作成する場合は、宣言またはオーバーライドしない属性またはサブフォルダを除外する必要があります。

パス・パターン

path引数でモデルのロケーションを指定するには、いくつかの方法があります。 いくつか例を挙げます。

すべてのトップレベル・モデル・セクション(topology、resourcesなど)をリストします:

top

セクション内の属性およびフォルダ(topology、resourcesなど)をリストします:

topology

フォルダ内のすべての属性およびフォルダをリストします:

resources:/JDBCSystemResource/JdbcResource

フォルダにセクションが指定されていない場合は、そのセクションが導出されて出力テキストに含まれます:

/JDBCSystemResource/JdbcResource

出力オプション

モデル・パスの出力テキストを制御するために使用できるコマンドライン・オプションがいくつかあります。 これらのオプションは一度に1つしか使用できません。 出力オプションが指定されていない場合は、指定されたパスの属性および即時サブフォルダがリストされます。

パスtopを使用して最上位セクションがリストされている場合、出力オプションは無視されます。

-attributes_only

このオプションでは、指定したパスの属性のみがリストされます。

-folders_only

このオプションでは、指定したパスの直接のサブフォルダのみがリストされます。

-recursive

このオプションは、指定されたパス内のすべてのサブフォルダを再帰的にリストします。 属性はリストされません。

-recursiveオプションの使用例を次に示します:

$ <wls-deploy-home>/bin/modelHelp.sh -oracle_home /tmp/oracle -recursive resources:/JDBCSystemResource

出力は次のとおりです。

Recursive sub-folders only for resources:/JDBCSystemResource

resources:
    JDBCSystemResource:
        'JDBC-1':
            JdbcResource:
                JDBCConnectionPoolParams:
                JDBCDataSourceParams:
                JDBCDriverParams:
                    Properties:
                        'Properties-1':
                JDBCOracleParams:
                JDBCXAParams:
            SubDeployment:
                'SubDeployment-1':
