ドメインの作成ツール

ドメインの作成ツールは、モデルとWLSTオフラインを使用してドメインを作成します。 このツールを使用するには、次に示すように、少なくともモデルのdomainInfoセクションでドメインの管理パスワードを指定する必要があります。 

domainInfo:
    AdminPassword: welcome1

前述のモデルを使用して、createDomainツールを実行し、作成するドメインのタイプとその作成場所を指定します。

$ weblogic-deploy\bin\createDomain.cmd -oracle_home c:\wls12213 -domain_type WLS -domain_parent d:\demo\domains -model_file MinimalDemoDomain.yaml

テンプレート定義のサーバーのみを含む空のドメインを作成することはあまり興味深いことではありませんが、この例では単にモデルのスパース性を強化するだけです。 ドメインの作成ツールを実行する場合、モデルはアーカイブ・ファイル内に指定するか、スタンドアロン・ファイルとして指定する必要があります。 アーカイブ・ファイルとモデル・ファイルの両方が指定されている場合は、アーカイブ外のモデル・ファイルがアーカイブ内のモデル・ファイルより優先されます。 アーカイブ・ファイルが指定されていない場合、ドメインの作成ツールはドメイン内のモデルのtopologyセクションのみを(domainInfoセクションを使用して)作成します。 これは、モデルのresourcesおよびappDeploymentsセクションがアーカイブからファイルを参照できるため、モデル定義のリソースおよびアプリケーションを使用してドメインを作成するには、モデルがアーカイブ内の何も参照していない場合でもアーカイブ・ファイルを指定する必要があるためです。 将来のある時点では、この制限は、実際に必要な場合にのみアーカイブを必要とするよう緩和される可能性があります。

ドメインの作成ツールは、3つのドメイン・タイプを理解: WLS、RestrictedJRFおよびJRF。 ドメイン・タイプを指定する場合、Oracle Homeはドメイン・タイプの要件を満たしている必要があります。 RestrictedJRFとJRFの両方に、FMWインフラストラクチャ(JRFとも呼ばれる)がインストールされたOracle Homeが必要です。 JRFドメインを作成する場合、RCUデータベース情報を引数としてcreateDomainスクリプトに指定する必要があります。 必要なパスワードの入力を求められることに注意してください。 必要に応じて、スクリプトをユーザー入力なしで実行するために、スクリプトの標準入力(stdinなど)にパイプできます。 たとえば、JRFドメインを作成するコマンドは次のようになります。 これには、コマンドを実行する前にRCUを実行しておく必要があります。 

$ weblogic-deploy\bin\createDomain.cmd -oracle_home c:\jrf12213 -domain_type JRF -domain_parent d:\demo\domains -model_file DemoDomain.yaml -rcu_db mydb.example.com:1539/PDBORCL -rcu_prefix DEMO [-rcu_db_user SYS]

ドメインの作成ツールでRCUを実行するには、-run_rcu引数を前のコマンド行に追加するだけで、RCUスキーマが自動的に作成されます。 ツールでRCUを実行すると、新しいスキーマを作成する前に、同じRCUプレフィクスを持つ競合するスキーマが自動的に削除されることに注意してください。

コマンドライン引数を使用するかわりに、モデルに接続情報を指定することもできます。 これは、複雑なデータベース接続文字列およびRACやOracle Autonomous Transaction Processing Cloud Serviceデータベースなどの追加パラメータを必要とするデータベースでは特に簡単です。 使用方法の詳細は、「モデルでのRCU接続情報の指定」を参照してください。

より複雑なドメインを作成するには、カスタム・ドメイン・タイプの作成が必要になる場合があります。 これは、ドメインにカスタム・テンプレートまたは他のOracle製品のテンプレートがある場合に便利です。 詳細は、「ドメイン・タイプ定義」を参照してください。

暗号化モデルの使用

WDT暗号化ツールを使用して暗号化されたパスワードがモデルまたは変数ファイルに含まれている場合、コマンドラインで-use_encryptionフラグを使用して作成中にパスワードを復号化し、暗号化が使用されていることをドメイン作成ツールに通知し、暗号化パスフレーズの入力を求めます。 データベース・パスワードと同様に、ツールは標準入力(stdinなど)からパスフレーズを読み取り、ユーザー入力なしでツールを実行できるようにすることもできます。

複数のモデルの使用

ドメインの作成ツールでは、「複数のモデルの使用」で説明されているように、複数のモデルの使用がサポートされます。

開発ドメインおよびboot.properties

WDTは、開発ドメインを作成するときに、ドメイン内の各サーバーに対してboot.propertiesファイルを作成するための便利な方法を提供します。 boot.propertiesファイルには、管理サーバーのユーザー名とパスワードの暗号化された値が含まれます。 管理サーバーまたは管理対象サーバーが起動されると、WebLogic Serverは資格証明のプロンプトをバイパスし、かわりにboot.propertiesファイルの資格証明を使用します。

ServerStartModeオプションがprodに設定されている場合、またはドメインProductionModeEnabledがtrueに設定されている場合、ドメインは本番モードになります。 これらの属性のデフォルト値はどちらも開発モードです。

boot.propertiesファイルは、WDTが実行されているマシンのドメイン・ホームに格納されます。 これは、サーバーごとに<domain_home>/servers/<server_name>/security/boot.propertiesとして格納されます。

次に、両方の属性が明示的に開発モードに設定されているモデルの例を示します。

domainInfo:
    AdminUserName: weblogic
    AdminPassword: welcome1
    ServerStartMode: dev
topology:
    Name: "my-domain"
    AdminServerName: "admin-server"
    ProductionModeEnabled: false
