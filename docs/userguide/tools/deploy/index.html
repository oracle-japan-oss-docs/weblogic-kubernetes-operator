Deploy Applications Tool

Deploy Applications Toolは、モデル、アーカイブおよびWLSTを使用して、WLSTオンライン・モードまたはオフライン・モードでアプリケーションおよびリソースを既存のWebLogic Serverドメインにデプロイします。 モデルからアプリケーションおよびリソースをデプロイする場合、デプロイ・ツールは主にモデルのresourcesおよびappDeploymentsセクションに焦点を当てます。 domainInfoおよびtopologyセクションには例外があり、これらの構成要素はアプリケーション関連とみなされます。 たとえば、サーバーのServerStartフォルダには、アプリケーションが適切に機能するために依存する可能性のあるサーバー環境(ノード・マネージャによって起動された場合)を変更するArgumentsおよびClassPath属性があります。 同様に、domainInfoセクションには、同様の理由でアプリケーションに関連する<DOMAIN_HOME>/libに配置されるJARファイルのリストが含まれます。

Deploy Applications Toolは、指定されたモデルの要素のみを追加または更新します。 以前のモデルに存在していた欠落している要素は削除されません。

WLSTオンライン・モードでは、ツールはアプリケーションと共有ライブラリの再デプロイの必要性を最小限に抑え、サーバーを再起動する必要があります。 これにはいくつかの方法があります:

  • モデルがすでにデプロイされているアプリケーションまたは共有ライブラリを参照している場合、ツールはバイナリを比較して再デプロイが必要かどうかを判断します。 共有ライブラリを使用するすべてのアプリケーションは、アプリケーションが変更されていない場合でも再デプロイする必要があるため、共有ライブラリの再デプロイには特にコストがかかります。
  • ナレッジ・ベースを参照して、変更時に再起動が必要な属性を判断します。 属性の再起動が必要な場合は、変更を適用する前に、ツールによって現在の値とモデル値が比較され、異なることが確認されます。

目的は、反復デプロイメントをサポートできるようにし、実行中のドメインに対して作業を行う際のサービス中断を最小限に抑えることです。

WLSTオフライン・モードでのDeploy Applications Toolの実行は、ドメインの作成ツールの実行と非常によく似ています。ドメインのロケーションとアーカイブ・ファイルを指定し、必要に応じてモデル・ファイルと変数ファイルを個別に指定します。 次に例を示します。 

$ weblogic-deploy\bin\deployApps.cmd -oracle_home c:\wls12213 -domain_home domains\DemoDomain -archive_file DemoDomain.zip -model_file DemoDomain.yaml -variable_file DemoDomain.properties

ツールをオンライン・モードで実行するには、WebLogic Server管理サーバーへの接続に必要な値を指定して、-admin_urlおよびadmin_userの引数を追加します。 次に例を示します。 

$ weblogic-deploy\bin\deployApps.cmd -oracle_home c:\wls12213 -domain_home domains\DemoDomain -archive_file DemoDomain.zip -model_file DemoDomain.yaml -variable_file DemoDomain.properties -admin_url t3://127.0.0.1:7001 -admin_user weblogic

通常どおり、ツールによってパスワードの入力が求められます(ツールの標準入力にパイピングして指定することもできます)。

WLSTオンライン・モードでツールを実行している場合、デプロイ操作では、変更を取得するためにサーバーの再起動またはドメインの再起動が必要になることがあります。 デプロイ操作では、ドメインが再起動されるまで操作を完了できない場合もあります。 Deploy Applications Toolをコールするスクリプトにこれらの条件を伝達するために、シェル・スクリプトには、これらの状態を伝達するためのゼロ以外の特別な終了コードが3つあります:

  • 103 - ドメイン全体を再起動する必要があります。
  • 104 - モデルの変更にはドメインの再起動が必要で、 -cancel_changes_if_restart_requiredが指定されているため、ドメインの変更は取り消されました。

暗号化モデルの使用

WDT暗号化ツールを使用して暗号化されたパスワードがモデルまたは変数ファイルに含まれている場合、コマンドラインで-use_encryptionフラグを使用して作成中にパスワードを復号化し、暗号化が使用されていることをDeploy Applications Toolに伝え、暗号化パスフレーズの入力を求めます。 データベース・パスワードと同様に、ツールは標準入力(stdinなど)からパスフレーズを読み取り、ユーザー入力なしでツールを実行できるようにすることもできます。

複数のモデルの使用

Deploy Applications Toolは、「複数のモデルの使用」で説明されているように、複数のモデルの使用をサポートしています。
