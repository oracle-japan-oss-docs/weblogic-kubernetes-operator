ドメイン・リソースの抽出ツール

WebLogic Kubernetes OperatorでのWDTの使用

ドメイン・リソースの抽出ツールを使用して、WebLogic Kubernetes Operatorで使用するドメイン・リソース・ファイルを作成できます。 これにより、ドメイン構成とKubernetesコンテナ構成を単一のモデル・ファイルで指定できます。

これは、ドメイン・リソース・ファイルにも反映する必要があるドメインの構成を変更する場合に特に便利です。 たとえば、クラスタをドメインに追加するには、WDTモデルのtopologyセクションに追加する必要があります。その後、新しいドメイン・リソース・ファイルを生成してKubernetesに適用できます。

WebLogic Kubernetes Operatorの詳細は、「こちら」を参照してください。

ドメイン・リソースの抽出ツールは、WDTリリース1.7.0以降で使用できます。

ドメイン・リソースの抽出ツールのコマンドラインの例を次に示します:

$ <wls-deploy-home>/bin/extractDomainResource.sh -oracle_home /tmp/oracle -domain_home /u01/mydomain -model_file /tmp/mymodel.yaml -variable_file /tmp/my.properties -domain_resource_file /tmp/operator/domain-resource.yaml

最も簡単なケースとして、ドメイン・リソースの抽出ツールによってスパース・ドメイン・ファイルが作成されます。 これは、モデルにkubernetesセクションがない場合、またはそのセクションが空の場合に生成される内容です。 

apiVersion: weblogic.oracle/v8
kind: Domain
metadata:
    name: DemoDomain
spec:
    domainHome: /u01/mydomain
    image: '--FIX ME--'
    imagePullSecrets:
    -   name: '--FIX ME--'
    webLogicCredentialsSecret: '--FIX ME--'
    configuration:
        model:
            domainType: WLS
    clusters:
    -   clusterName: mycluster
        replicas: 2
    -   clusterName: mycluster3
        replicas: 4

この例では、domainHomeの値がコマンドラインからextractDomainResourceスクリプトの入力パラメータとして設定されています。 kindおよびnameは、モデルのトポロジ・セクションまたはデフォルトのbase_domainから導出されたドメイン名に設定されています。 クラスタ・エントリはモデルのトポロジ・セクションから取得され、そのレプリカ数は各クラスタのサーバー数から導出されました。

ユーザーは、ドメイン・リソース出力で--FIX ME--によって識別されるイメージおよびシークレット情報を入力する必要があります。

--FIX ME--値の事前移入を含む、より高度な構成の場合、ユーザーはWDTモデルのkubernetesセクションに移入でき、これらの値は結果のドメイン・リソース・ファイルに表示されます。 このモデル・セクションは、いくつかの値をオーバーライドして結果に追加します。 

kubernetes:
    metadata:
        name: myName
        namespace: myNamespace
    spec:
        image: 'my.repo/my-image:2.0'
        imagePullSecrets:
            WEBLOGIC_IMAGE_PULL_SECRET_NAME:
        webLogicCredentialsSecret:
            name: '@@PROP:mySecret@@'
        configuration:
            model:
                domainType: 'WLS'
            secrets: [
              'secret1',
              'secret2'
            ]
        serverPod:
            env:
                USER_MEM_ARGS:
                    value: '-XX:+UseContainerSupport -Djava.security.egd=file:/dev/./urandom'
                JAVA_OPTIONS:
                    value: '-Dmydir=/home/me'

この例では、@@PROP:mySecret@@を使用して、コマンドラインで指定された変数ファイルからwebLogicCredentialsSecretの値を取得します。 これは、モデルのkubernetesセクションの任意の値を使用して実行できます。 モデル変数の使用の詳細は、「こちら」を参照してください。

この例では、結果のドメイン・リソース・ファイルに次のものが含まれます:

apiVersion: weblogic.oracle/v8
kind: Domain
metadata:
    name: myName
    namespace: myNamespace
spec:
    image: 'my.repo/my-image:2.0'
    imagePullSecrets:
    -   name: WEBLOGIC_IMAGE_PULL_SECRET_NAME
    webLogicCredentialsSecret:
        name: WEBLOGIC_CREDENTIALS_SECRET_NAME
    serverPod:
        env:
        -   name: USER_MEM_ARGS
            value: '-XX:+UseContainerSupport -Djava.security.egd=file:/dev/./urandom'
        -   name: JAVA_OPTIONS
            value: '-Dmydir=/home/me'
    domainHome: /u01/mine/domain
    configuration:
        model:
            domainType: WLS
        secrets:
        -   secret1
        -   secret2
    clusters:
    -   clusterName: mycluster
        replicas: 2
    -   clusterName: mycluster3
        replicas: 4

WDTモデルのspec/serverPod/envおよびその他のリスト・セクションの構文は、ターゲット・ファイルの構文とは異なります。 WDTツールではハイフン付きリスト構文が認識されないため、これらの要素は他のモデル・リストと同様に指定されます。

モデルのkubernetes/specセクションでクラスタが指定されている場合、それらのクラスタはドメイン・リソース・ファイルで構成され、topologyセクションのクラスタは無視されます。

WDTモデルの値がNever for spec/imagePullPolicyの場合、imagePullSecretsのデフォルト値は追加されません。

WebLogic Kubernetes Operatorでサポートされているセクションおよび変数の完全なリストは、「こちら」を参照してください。

ドメイン・リソースの抽出ツールは、metadata、serverPod、specなど、これらのセクションのサブセットをサポートしています。

「モデル・ヘルプ・ツール」を使用して、モデルのkubernetesセクションで使用できるフォルダおよび属性を決定できます。 たとえば、次のコマンドでは、specフォルダ内のフォルダおよび属性がリストされます: 

<wls-deploy-home>/bin/modelHelp.sh -oracle_home /tmp/oracle kubernetes:/spec

kubernetesセクションのコンテンツは、ドメインの検出ツールによってモデルが検出されたときには生成されません。
