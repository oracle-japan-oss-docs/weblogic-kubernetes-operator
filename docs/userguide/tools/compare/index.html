モデルの比較ツール

ドメイン・モデルを使用する場合、異なるモデル間の違いを把握しておくと便利な場合があります。 モデルの比較ツールは、2つのモデル・ファイルを比較し、それらの違いを示すモデルを生成します。

モデルの比較ツールを使用するには、正しい引数を指定してcompareModelシェル・スクリプトを実行します。 有効な引数のリストを表示するには、-helpオプションを指定して(または引数を指定せずに)シェル・スクリプトを実行し、使用方法を確認します。

たとえば、次のモデルを比較します。

2つのモデル・ファイルを比較するには、次のようにツールを実行します:

$ weblogic-deploy\bin\compareModel.cmd -oracle_home c:\wls12213 new_model.yaml old_model.yaml

ツールの出力は次のようになります:

Comparing Models: new=/tmp/model2.yaml vs old=/tmp/model1.yaml

Differences between new model and old model:

resources:
    JMSSystemResource:
        MyJmsModule:
            JmsResource:
                ConnectionFactory:
                    '!MDBConnectionFactory':
    '!WebAppContainer':
appDeployments:
    Application:
        '!myear2':
        yourear:
            SourcePath: /home/johnny/dimtemp23/sample_app_stage/wlsdeploy/applications/sample_app2.ear
            ModuleType: ear
            Target: [ 'cluster-2', 'cluster-1' ]
        myear:
            Target: [ 'cluster-2', 'cluster-1' ]
topology:
    ServerTemplate:
        'cluster-1-template':
            '!ServerStart':
        'cluster-2-template':
            Cluster: 'cluster-2'
            ListenPort: 8001
            ServerStart:
                Arguments: [ '-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=6006' ]
            JTAMigratableTarget:
                StrictOwnershipCheck: True
                Cluster: 'cluster-2'
    Cluster:
        'cluster-2':
            DynamicServers:
                ServerTemplate: 'cluster-2-template'
                ServerNamePrefix: 'managed-server'
                DynamicClusterSize: 2
                MaxDynamicClusterSize: 3
                CalculatedListenPorts: False


1. Model Path: resources-->JMSSystemResource-->MyJmsModule-->JmsResource-->UniformDistributedQueue-->MyUniformDistributedQueue-->ResetDeliveryCountOnForward does not exist in new model but exists in previous model

2. Model Path: appDeployments-->Application-->myear-->ModuleType does not exist in new model but exists in previous model

新しいモデルと古いモデルの比較:

  1. topologyセクションにcluster-2およびcluster-2-templateが追加されました。
  2. topologyセクションのcluster-1-templateのServerStartが削除されました。
  3. appDeploymentsセクションに、yourearとmyearの2つのアプリケーションをデプロイしました。
  4. appDeploymentsセクションのアプリケーションmyearが削除されました。
  5. resourceセクションのMyJmsModule JMSモジュールからMDBConnectionFactoryが削除されました。
  6. resourceセクションのWebAppContainerが削除されました。
  7. MyUniformDistributedQueueから属性ResetDeliveryCountOnForwardが削除されました。
  8. アプリケーションmyearのデプロイメント・ターゲットを変更しました。
  9. アプリケーションmyearの属性'ModuleType'を削除しました。

!は、モデルから非属性キー要素を削除するための表記法です。 欠落している属性は結果のモデルから省略されますが、メッセージとして出力に表示されます。 属性値がリストの場合、リスト内の個々のアイテムがすべて同じでも順序が異なる場合、属性は異なるものとしてカウントされます。

2つのモデル・ファイルを比較し、出力をファイルに生成するには、次のようにツールを実行します:

$ weblogic-deploy\bin\compareModel.cmd -oracle_home c:\wls12213 -output_dir c:\cm-output ew_model.yaml old_model.yaml

The following files will be written to the directory:

diffed_model.json
diffed_model.yaml
compare_model_stdout
