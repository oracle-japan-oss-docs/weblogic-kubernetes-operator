ドメイン・タイプ定義

WebLogic Deploy Toolingには、拡張可能なドメイン・タイプ・システムがあります。 3つの組込みドメイン・タイプ(WLS、RestrictedJRFおよびJRF)は、WLSDEPLOY_HOME/lib/typedefsディレクトリの同じ名前のJSONファイルで定義されています。 たとえば、JRFドメイン・タイプは、次に示すように、WLSDEPLOY_HOME/lib/typedefs/JRF.jsonファイルで同様のコンテンツを使用して定義されます。

このファイルは、ドメインの作成に使用するテンプレート、ターゲットにするサーバー・グループ、および作成するRCUスキーマをすべて、インストールされているWebLogic Serverのバージョンに基づいて示します。

新しいドメイン・タイプを定義するには、WLSDEPLOY_HOME/lib/typedefsディレクトリに同じ構造の新しいJSONファイルを作成します。

別の方法として、「カスタム構成」ディレクトリ$WDT_CUSTOM_CONFIG/typedefsにこのファイルを作成することもできます。

たとえば、12.2.1.3のSOAドメイン・タイプを定義するには、次のような内容のtypedefs/SOA.jsonファイルを追加します。

{
    "name": "SOA",
    "description": "SOA type domain definitions",
    "versions": {
        "12.2.1.3": "SOA_12213"
    },
    "definitions": {
        "SOA_12213": {
            "baseTemplate": "Basic WebLogic Server Domain",
            "extensionTemplates": [
                "Oracle SOA Suite"
            ],
            "serverGroupsToTarget": [ "JRF-MAN-SVR", "WSMPM-MAN-SVR",  "SOA-MGD-SVRS" ],
            "rcuSchemas": [ "STB", "WLS", "MDS", "IAU", "IAU_VIEWER", "IAU_APPEND", "OPSS", "UCSUMS", "SOAINFRA" ]
        }
    }
}

新しいドメインtypedefファイルが存在するようになったら、必要なコンポーネントがインストールされているOracle Homeを参照して、新しいドメイン・タイプ名をcreateDomainスクリプトに指定します。 12.2.1より前のバージョンの場合は、-wlst_path引数を使用して、適切なWLSTシェル・スクリプトが存在する製品ホームを指す必要があります。たとえば、SOA 12.1.3の場合は、ツールがSOAドメインの適切な環境でWLSTシェル・スクリプトを使用するように-wlst_path <ORACLE_HOME>/soaを追加します。 12.2.1以降では、標準の<ORACLE_HOME>oracle_common/common/binディレクトリにあるWLSTシェル・スクリプトによってすべてのコンポーネントがOracle Homeに自動的にロードされるため、これは必要なくなりました。 新しいドメイン・タイプを使用して、次のコマンドを実行してRCUを実行し、そのすべてのリソースおよびアプリケーションがデプロイされたSOAドメインを作成します。 

 weblogic-deploy\bin\createDomain.cmd -oracle_home d:\SOA12213 -domain_type SOA -domain_parent d:\demo\domains -model_file DemoDomain.yaml -archive_file DemoDomain.zip -variable_file DemoDomain.properties -run_rcu -rcu_db mydb.example.com:1539/PDBORCL -rcu_prefix DEMO [-rcu_db_user SYS]

カスタム拡張テンプレート

customExtensionTemplates属性を使用して、ドメインに適用するカスタム拡張テンプレートを指定できます。 これらは絶対ファイル・パスとして指定する必要があり、トークンを使用できます。 

{
    "name": "MyCustom",
    "description": "My custom type domain definitions",
    "versions": {
        "12.2.1.3": "My_12213"
    },
    "definitions": {
        "My_12213": {
            "baseTemplate": "Basic WebLogic Server Domain",
            "extensionTemplates": [ ],
            "customExtensionTemplates": [
                "/user/me/templates/my-template.jar",
                "@@ORACLE_HOME@@/user_templates/other-template.jar"
            ],
            "serverGroupsToTarget": [ "MY-MAN-SVR" ],
            "rcuSchemas": [ ]
        }
    }
}

カスタム・テンプレートに管理対象サーバーをターゲットにするサーバー・グループがある場合は、前述の例のMY_MAN_SVRのように、serverGroupsToTarget属性で指定する必要があります。

以前のバージョンのWebLogic Serverでのターゲット指定

12.2.1より前のWebLogic Serverバージョンのテンプレートでは、リソースを正しいクラスタおよびサーバーに正しくターゲット指定するために、applyJRF WLSTコマンドの使用が必要になる場合があります。 WebLogic Deploy Toolingのデフォルトの動作では、extensionTemplatesリストにJRFテンプレートが含まれている場合にのみapplyJRFが起動されます。

カスタム・タイプ定義ファイルでは、テンプレートの追加後にapplyJRFの起動が必要になる場合があります。 これを行うには、次の例に示すように、targeting属性をAPPLY_JRFに設定します: 

{
    "name": "MyCustom",
    "description": "My custom type domain definitions",
    "versions": {
        "10.3.6": "My_11G"
    },
    "definitions": {
        "My_11G": {
            "baseTemplate": "@@WL_HOME@@/common/templates/wls/wls.jar",
            "extensionTemplates": [
                "/user/me/templates/my-template.jar"
             ],
            "targeting": "APPLY_JRF",
            "serverGroupsToTarget": [ "MY-MAN-SVR" ],
            "rcuSchemas": [ ]
        }
    }
}

targeting属性は、WebLogic Serverバージョン12.2.1以上では無効です。
