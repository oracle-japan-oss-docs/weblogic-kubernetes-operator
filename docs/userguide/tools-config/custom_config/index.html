カスタム構成

WDTを使用すると、事前インストール済の型定義、モデル・フィルタ、変数インジェクタおよびターゲット環境を作成または拡張できます。 WDT 1.10.0以降では、これらの追加構成ファイルを$WLSDEPLOY_HOME/libディレクトリの外部に格納できます。 これにより、WDTインストールを新しいバージョンに移動またはアップグレードした場合でも、ファイルを所定の場所に残しておくことができます。

別の構成ディレクトリを使用するには、WDT_CUSTOM_CONFIG環境変数を構成に使用するディレクトリに設定します。 次に例を示します。 

$ export WDT_CUSTOM_CONFIG=/etc/wdtconfig

カスタマイズされた構成ファイルは、$WLSDEPLOY_HOME/libディレクトリの下と同じ方法で名前を付けて編成する必要があります。 次に例を示します。 

/etc/wdtconfig
    injectors
        *.json  (injector files)
    typedefs
        *.json  (typedef files)
    targets
        my-target
            target.json
            *.py  (filter files)
    model_filters.json
    model_variable_injector.json
    variable_keywords.json

これは、構成可能なファイルの完全なセットです。 追加する必要があるのは、作成または拡張したファイルのみです。 各構成タイプの詳細は、次を参照してください:

  • ツールのプロパティ・ファイル
  • モデル・フィルタ
  • 「型定義」 (次の「型定義の拡張」の例を参照。)
  • 変数インジェクション
  • 「モデルの準備ツール」、「ターゲット環境」を参照してください。

WDTツールは、$WDT_CUSTOM_CONFIGの下の各構成ファイルを検索し(指定されている場合)、次に$WLSDEPLOY_HOME/libの下を検索します。

Example: 型定義の拡張

WLS型定義を拡張するには、次のステップに従います:

  • /etc/wdtconfigなどのカスタム構成に使用するディレクトリを作成します。
  • そのディレクトリを指すようにWDT_CUSTOM_CONFIG環境変数を定義します。
  • ファイル$WLSDEPLOY_HOME/lib/typedefs/WLS.jsonを$WDT_CUSTOM_CONFIG/typedefsディレクトリにコピーし、名前を変更します(例: MY_WLS.json)。
  • 必要な変更を加えてMY_WLS.jsonを編集します。
  • 新しい型定義の名前を参照するツールを実行します。次に例を示します:
$ createDomain.cmd -oracle_home /wls12213 -domain_type MY_WLS ...
